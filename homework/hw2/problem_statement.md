# Домашнее задание

## Качество кода и богатая модель

Цель: устранить примитивные типы через внедрение Value Objects и применить принципы SOLID для разделения ответственностей в доменной модели.

  

## Описание/Пошаговая инструкция выполнения домашнего задания:

## Инструкция:

  

1.  Борьба с примитивами (Value Objects): В вашем домене выделите 3-4 концепции, которые обычно реализуют через примитивы (string, int, decimal). Спроектируйте и реализуйте в коде один из них как полноценный Value Object (например, Money, Email, DeliveryAddress).

  

-   Требования к VO: неизменяемость (immutable), валидация в конструкторе, реализация структурного равенства.

  

2.  Unit-тесты для VO: Напишите unit-тесты, которые проверяют логику создания и сравнения вашего Value Object.

  

3.  Анализ и рефакторинг (SOLID): Найдите в своём коде (или придумайте) «плохой» класс, нарушающий принципы SOLID (например, God Object Order с методами для расчета, оплаты и доставки).

  

4.  Применение SOLID: Опишите, какие принципы нарушены. Предложите и напишите (можно псевдокодом) рефакторинг этого класса: разделите ответственности (SRP), выделите абстракции (DIP/ISP) и используйте созданный ранее Value Object.

## Критерии оценки:

Статус «Принято», если:

1.  Моделирование VO: Строгая неизменяемость (immutability) и наличие валидации (защиты инвариантов) при создании Value Object.
2.  Тестируемость: Наличие unit-тестов, проверяющих именно бизнес-правила VO, а не просто работу геттеров.
3.  Архитектурное мышление: Понимание нарушений SOLID и жизнеспособность предложенного рефакторинга.

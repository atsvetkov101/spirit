# Система Спирит - система управления разъездными сотрудниками

## 1. Выбор и анализ домена: выбран домен "Заявки и координаты" - система Спирит

Список доменов компании:
- Заявки и координаты - система Спирит
- Учет заказчиков и объектов обслуживания - система Creatio
- Учет/регистрация исполнителей - система Creatio
- Учет/регистрация входящих заявок от заказчиков - система Creatio
- Аутентификация и доступ к корп. сети - Active Directory
- Учет оборудования (склады компании и виртуальные склады исполнителей) - система 1С
- Передача отчетной документации(отправка актов заказчикам) - отдельная система компании
- Расчет перемещений исполнителей и расчет компенсаций - отдельная система компании
- Уведомления - отдельная система компании (использование Firebase и мессенджеров)

## 2. Список поддоменов системы Спирит. Домен "Заявки и координаты"

- Core: Заявки

- Core: Координаты

- Generic: Сотрудники

- Supporting: Оборудование

- Generic: Уведомления

- Supporting: Отчетные документы

Обоснование: Первым Core-субдоменом является субдомен "Заявки". Основная деятельность инженеров - выполнение заявок. Основная деятельность руководителей - настройка правил выполнения заявок. Получение вспомогательных документов, фотографий, выполняется в процессе работы над заявкой. После выполнения, закрытия заявки, формируется акт выполненных работ и прикладывается к списку файлов заявки. Полный набор отчетных документов, сформированный в процессе работы над заявкой, формируется в системе Спирит и отправляется заказчику через отдельную систему. Создание заявок осуществляется в другой системе, с помощью регистрации обращений от заказчиков. Назначением заявок на исполнителя занимается отдельная система. Интеграция системы Спирит с системой создания и назначения заявок является самой важной интеграцией, т.к. влияет на время взятия заявки в работу и на время выполнения заявки. Таким образом именно процесс выполнения заявки исполнителем является основным бизнес-процессом, который автоматизирует система Спирит.  
Второй core-субдомен - "Координаты". В целях разработки системы Спирит зафиксировано, что сбор информации о перемещении исполнителей является 2-й важной функцией системы Спирит. Информация собирается с мобильных устройств исполнителей и аккумулируется в БД.

| ПодДомен | Ограниченный контекст |
| :--- | :--- |
| Заявки| Учет заявок |
| Заявки| Работа над заявками |
| Координаты | Координаты |
| Сотрудники | Сотрудники |
| Оборудование | Оборудование |
| Уведомления | Уведомления |
| Отчетные документы | Отчетные документы |

**Трактование понятий**
"Заявка" в ограниченном контексте "Учет заявок" характеризуется как сущность, которая имеет свойства:
- Идентификатор
- Дата создания
- Статус
- Объект обслуживания
- Назначенный Исполнитель
"Заявка" в ограниченном контексте "Работа над заявками" характеризуется как сущность, которая имеет свойства:
- Сервис и Категория - определяют правила выполнения заявки
- Чек-лист
- Отчетные документы
- Координаты исполнителя в процессе выполнения заявки
- Оборудование
"Заявка" в ограниченном контексте "Координаты" - это набор координат исполнителя, в процессе работы над заявкой.
"Заявка" в ограниченном контексте "Оборудование" - это список оборудования, которое было установлено или демонтировано в процессе работы над заявкой
"Заявка" в ограниченном контексте "Уведомления" - это набор уведомлений, которые были отправлены пользователю по данной заявке. 
"Заявка" в ограниченном контексте "Отчетные документы" - это набор документов, полученных от исполнителя и сгенерированных по результатам работы над заявкой.

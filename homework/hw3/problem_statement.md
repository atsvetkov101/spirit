# Домашнее задание

## Проектирование Агрегатов и их жизненный цикл

Цель: спроектировать корень агрегата с защитой инвариантов, реализовать доменные интерфейсы репозитория и фабрики для управления жизненным циклом агрегата.

## Описание/Пошаговая инструкция выполнения домашнего задания:

## Инструкция:

1.  Проектирование Агрегата:

-   Спроектируйте и реализуйте в коде Агрегат Order (Заказ). Корнем агрегата является сам Order, а OrderLine (строка заказа) — его внутренней сущностью.
-   Используйте Value Object Money из прошлого ДЗ.
-   Реализуйте бизнес-метод AddItem(product, quantity), который защищает инвариант: «в заказ нельзя добавить больше 10 позиций».
-   Убедитесь, что внешний код не может напрямую изменить коллекцию строк заказа в обход корня.

2.  Проектирование Репозитория: Напишите интерфейс IOrderRepository. Названия методов должны соответствовать Единому языку (например, FindByCustomer(customerId) ), а не CRUD-операциям. Интерфейс должен работать только с корнем агрегата Order .
    
3.  Проектирование Фабрики: Создайте доменную Фабрику (класс или статический метод) для создания Агрегата Order. Фабрика должна принимать начальные данные и возвращать объект Order в гарантированно валидном состоянии.
    
4.  Тестирование связки: Напишите In-Memory реализацию IOrderRepository (на основе List или Dictionary). Напишите unit-тест, который с помощью Фабрики создает Заказ, сохраняет его через In-Memory репозиторий, а затем извлекает и проверяет его состояние.

## Критерии оценки:

Статус «Принято», если:

1.  Инкапсуляция: Отсутствие публичных сеттеров для коллекций внутри агрегата (защита внутренних сущностей).
2.  Бизнес-правила: Корректная реализация проверки инвариантов внутри корня агрегата и Фабрики.
3.  Инфраструктура: Отсутствие протечек технических деталей БД в интерфейс репозитория; успешное прохождение unit-теста с In-Memory  
    хранилищем.
